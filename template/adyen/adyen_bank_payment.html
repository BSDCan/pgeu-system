Invoice payment

{%if available%}

Note that use of this method for payment is discouraged. Due to the unreliable nature and slow processing, this method should only be used if no other methods are available. If possible, a bank transfer using Trustly or a payment using creditcard should be used instead.

Our IBAN payment management is handled by our payment processing company, Adyen. The payment should be made to their account, using the reference and instructions received after following the link below.

Due to the asynchronous nature of IBAN transfers, they are only allowed for invoices that are not close to their expiry time. The following special rules apply to these payments:

  • Payments must be made exactly as instructed on the payment page. There is no manual handling in the process, so if the reference is incorrect, or if the payment is not done in full, it will not be matched by the system.
  • Any transfer fees must be pre-paid (set for sender pays). If any fees are deducted from the payment, it will not match in the system, and thus fail.
  • The payment must reach our systems before the invoice is canceled. This includes delays introduced by sending bank, receiving bank and payment processor. This usually requires 3-5 days, so we recommend that you initiate the payment as soon as possible.
  • If any of these requirements are not fulfilled before the invoice is automatically canceled, the payment will be automatically returned to the sending account, minus transaction fees. We do not have the ability to override this behaviour.
  • Payments done using IBAN transfer are completely invisible to our systems until they are fully completed, so we have no ability to help figure out what went wrong if something did. We see no information at all about failed payments.
